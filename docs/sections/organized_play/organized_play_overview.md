# Organized Play Overview

## Purpose

This document defines how Organized Play is executed at A&H Games.

It establishes:

* Authority during events
* Enforcement expectations
* Event execution standards
* Consistency across employees and locations

Organized Play must be run the same way regardless of who is running the event or which location it is held at.

---

## Scope

This document applies to:

* All organized play events
* All employees running or assisting with events
* All locations

This document does not replace game rules or official tournament policies.

---

## Core Principles

* Events must be run consistently across all locations
* Rules must be enforced clearly and immediately
* Employees may not ignore issues
* Players must be treated fairly and equally
* Events must start and run on time

---

## Authority Structure

### Event Authority

* The event judge or event lead has primary authority during the event
* All rulings must go through the judge/event lead first
* Players may not bypass this and go directly to management

### Escalation

* Managers have final authority
* Issues must be escalated to a manager when:

  * A ruling is disputed and cannot be resolved
  * A player must be removed from the event
  * A ban is being considered
  * The employee is uncertain how to proceed

### Final Decisions

* Manager decisions are final
* Once a decision is made, it is not debated

---

## Employee Responsibilities

Employees running events must:

* Be present and actively managing the event
* Enforce all store policies and event rules
* Make rulings when required
* Maintain control of the event environment
* Escalate issues when necessary

Employees may not:

* Ignore rule violations
* Allow players to self-regulate disputes
* Delay enforcement to avoid discomfort

Failure to enforce rules is a policy violation.

---

## Policy Enforcement During Events

All store policies apply during organized play.

See:

* [Customer Conduct](../policies/customer_conduct.md)
* [Family-Friendly Environment](../policies/family_friendly.md)
* [Store Safety](../policies/store_safety.md)

### Enforcement Expectations

* Violations must be addressed immediately
* Employees must follow the enforcement steps defined in policy
* Players may be removed from the event if behavior does not improve
* Severe behavior may result in immediate removal

### Removal from Event

* A player removed from an event:

  * Is removed from the tournament immediately
  * Does not receive a refund
* Do not argue or negotiate once removal is decided

---

## Event Execution Standards

### Start Time

* Events must start at the scheduled time
* Do not delay events for late players

### Late Registration

* Late registration is allowed only if manageable
* Late players:

  * Receive losses for any rounds missed
  * Must be added without disrupting the event

If late registration creates disruption, it must be denied.

---

## Minimum Player Count

* Some events require a minimum number of players
* This is determined per event

If minimum requirements are not met:

* The event may not run
* Follow any game-specific requirements when applicable

---

## Event Staffing

* Events may be run by non-managers
* The assigned employee is responsible for the event

### Playing in Events

* Employees may not play in events they are running unless explicitly approved
* If approved:

  * Event integrity must not be compromised
  * The employee must still fulfill all responsibilities

---

## Consistency Across Locations

* Events must be run the same way at all locations
* This includes:

  * Structure
  * Enforcement
  * Player expectations

Employees may not create location-specific rules or exceptions.

## Summary

* Judge/event lead runs the event
* Manager has final authority
* Policies must be enforced immediately
* Events start on time

This document is enforceable. All employees are expected to follow it.
