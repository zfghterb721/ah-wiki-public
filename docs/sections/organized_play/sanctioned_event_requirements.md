# Sanctioned Event Requirements

## Purpose

This document defines the required standards for running sanctioned events at A&H Games.

Sanctioned events must follow all publisher, distributor, and system requirements.

Failure to comply may result in:

* Loss of event privileges
* Loss of product allocation
* Disciplinary action against the store

---

## Scope

Applies to:

* All sanctioned events
* All games using official tournament systems
* All employees scheduling or running events

---

## Definition

A sanctioned event is any event that:

* Is run through an official publisher system
* Reports results to a publisher or governing body
* Uses official prize support or kits

This includes (but is not limited to):

* Magic: The Gathering (WPN)
* Pokémon League, Challenges, and Cups
* Flesh and Blood Armory and special events
* Yu-Gi-Oh! OTS events
* Lorcana official events and championships

---

## System Requirements

Sanctioned events must:

* Be created in the correct official system
* Use the correct store account and location
* Be run entirely within that system
* Have accurate event details entered

Do not:

* Run sanctioned events outside official systems
* Create duplicate or incorrect events
* Use the wrong location or account

---

## Scheduling Requirements

### Advance Scheduling

Some events require scheduling in advance.

* Follow all publisher-specific timelines
* Do not schedule events late if advance notice is required

Example:

* Pokémon League Challenge and Cup events must be scheduled in advance through the official system

Failure to schedule correctly may invalidate the event.

---

## Event Accuracy

Event details must be correct before the event begins.

This includes:

* Format
* Start time
* Location
* Event type

Incorrect event setup may:

* Invalidate results
* Prevent reporting
* Violate publisher requirements

---

## Minimum Player Requirements

Some sanctioned events require a minimum number of players.

* Verify requirements before the event begins
* Do not run events that do not meet minimum requirements when prohibited

Example:

* Certain championship events require a minimum number of players to award prizes

If requirements are not met:

* Follow publisher instructions (including cancellation if required)

---

## Judge and Staff Requirements

Some events require a designated judge or organizer.

* Verify requirements before the event
* Ensure required roles are filled

Example:

* Certain championship-level events require a non-playing judge

If required staff are not present:

* Do not run the event if prohibited

---

## Reporting Requirements

Sanctioned events must be properly reported.

This includes:

* Entering all players into the system
* Recording match results accurately
* Submitting required reports within deadlines

Example:

* Pokémon League attendance must be reported monthly

Failure to report correctly may:

* Invalidate the event
* Affect store standing with the publisher

---

## Player Eligibility

Players must meet all event requirements.

* Must have required accounts or IDs
* Must be registered correctly in the system
* Must comply with event rules

Do not allow:

* Unregistered players in sanctioned events when prohibited
* Incorrect or placeholder entries

---

## Prize Support Compliance

Sanctioned prize support must be distributed correctly.

* Follow all publisher instructions for prize distribution
* Do not alter required prize structures when prohibited
* Do not substitute or withhold official prizes

Additional prizes:

* May be added unless prohibited
* Must be announced before the event

Cash prizes:

* Not allowed unless explicitly permitted

---

## Early Release Compliance

Sanctioned events tied to product release must follow early release rules.

* Do not distribute or sell product early unless allowed
* Follow all prerelease and early access rules

Violations may result in loss of privileges.

---

## Event Integrity

Sanctioned events must be run fairly and consistently.

* Apply rules equally to all players
* Do not allow favoritism
* Ensure accurate pairings and results

Do not:

* Modify results
* Skip rounds or reporting steps
* Alter event structure outside allowed rules

---

## Cancellation Requirements

If an event cannot meet required standards:

* Do not run the event if prohibited
* Follow publisher instructions for cancellation
* Return or handle materials as required

---

## Employee Responsibility

Employees running sanctioned events must:

* Understand all applicable requirements before the event
* Follow all system and reporting rules
* Verify compliance before, during, and after the event

Do not:

* Assume requirements
* Skip steps
* Ignore publisher rules

---

## Summary

* Sanctioned events must follow all official requirements
* Use the correct system and setup
* Meet player and staffing requirements
* Report events accurately and on time
* Follow prize and early release rules

Failure to comply is not acceptable.
