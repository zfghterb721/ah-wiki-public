# Event Types

All events must be categorized and run according to their defined structure.

Employees must understand the type of event they are running. Different event types have different expectations, but all must follow core event rules.

Employees may not treat all events the same.

---

## Core Rules

- Every event must have a **defined type before registration begins**
- Event type determines:
  - Structure
  - Rules enforcement level
  - Prize support
  - Player expectations
- Employees may not:
  - Change event type mid-event
  - Mix rules between event types
  - Create undefined or “custom” event types without manager approval

---

## Primary Event Types

### Tournament Events

Structured, competitive events with strict enforcement.

Examples:
- Weekly tournaments
- Competitive constructed events

#### Requirements

- Fixed round structure
- Strict time enforcement
- Standings-based results
- Full enforcement of rules and penalties

#### Employee Expectations

- Enforce rules strictly
- Maintain tight control of timing
- Resolve disputes immediately
- Monitor for cheating and slow play

---

### Casual Events

Structured but lower-pressure events focused on participation.

Examples:
- Casual play nights
- Learn-to-play events
- Introductory formats

#### Requirements

- Defined structure (even if simplified)
- Flexible pacing when appropriate
- Clear expectations for participation

#### Employee Expectations

- Maintain structure while allowing a relaxed environment
- Enforce rules, but focus on clarity and guidance
- Prevent disruption or abuse of relaxed standards

Employees may not:

- Allow casual events to become unstructured or chaotic
- Ignore rule violations entirely

---

### Prerelease Events

Product-based events with fixed capacity and materials.

Examples:
- TCG prereleases using prerelease kits

#### Requirements

- Capacity limited by product availability
- Each player receives defined product
- Event structure defined by product/event guidelines

#### Employee Expectations

- Enforce strict registration limits
- Distribute product accurately
- Prevent product misuse or redistribution
- Maintain timing and structure

Employees may not:

- Overfill events beyond product limits
- Redistribute product outside event rules

---

### Draft Events

Structured events where players build decks from opened product.

#### Requirements

- Controlled drafting process
- Equal distribution of product
- Defined drafting procedure

#### Employee Expectations

- Clearly explain draft process before starting
- Monitor drafting to prevent errors or abuse
- Ensure all players follow the same procedure

Employees may not:

- Allow players to deviate from draft process
- Allow unequal product distribution

---

### League Events

Ongoing events that span multiple sessions.

#### Requirements

- Defined tracking system (points, standings, etc.)
- Clear rules for participation across sessions

#### Employee Expectations

- Maintain accurate records
- Enforce participation rules consistently
- Prevent manipulation of standings

Employees may not:

- Allow untracked participation
- Modify standings without cause

---

### Special Events

Events that do not fit standard categories but are pre-approved.

Examples:
- Store-hosted special tournaments
- Unique formats
- Promotional events

#### Requirements

- Must be defined and approved by management before announcement
- Must have clearly documented rules and structure

#### Employee Expectations

- Follow defined structure exactly
- Do not improvise or modify rules

Employees may not:

- Create or run special events without approval
- Modify approved structure during the event

---

### Painting Competitions

Submission-based events with external voting systems.

#### Requirements

- Must follow **painting_competitions.md** exactly
- Controlled submission system
- Controlled voting process

#### Employee Expectations

- Follow all submission, posting, and voting procedures
- Maintain neutrality
- Prevent manipulation

Employees may not:

- Accept submissions outside the defined system
- Modify voting procedures
- Influence results

---

## Event Type Selection

Before registration begins, employees must confirm:

- Event type
- Structure
- Rules enforcement level
- Prize support

Employees may not begin an event without a clearly defined type.

---

## Mixing Event Types

Event types must remain distinct.

Employees may not:

- Combine casual and tournament structures
- Apply relaxed rules to competitive events
- Apply strict penalties to casual events without cause

---

## Non-Negotiable Rules

- No undefined event types
- No switching event types mid-event
- No mixing structures between types
- No improvisation of event formats

Failure to follow event type definitions is a performance issue.
